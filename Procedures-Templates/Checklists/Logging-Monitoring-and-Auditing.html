Sourced from: GCP Checklist 6 — Logging ,Monitoring and Alerting (maintaining reliability)

  • Configure Audit collection project
  • Configure a test project to use to validate the configuration of your data-access audit collection before propagating to developer and production projects,
  • Apply appropriate IAM controls to restrict who can access the audit logs
  • Export logs for longer term retention
  • Design aggregated exports on which your organization can filter and export the data for future analysis
  • Configure log sinks before you start receiving logs
  • Follow the best practices for common logging export scenarios
  • Make sure to use exclusion filters to exclude logging data that isn’t useful
  • Set appropriate IAM controls against the export sink destination 
  • Instrument your applications to provide metrics Configure Stackdriver for monitoring and alerting
