Data Lifecycle Checklist

 

This checklist is to be used to ensure that all aspects of the Data Lifecycle have been properly documented and implemented for client projects. (Ingested, Transformed, Curated, Queried, Explored, Analyzed and visualized, Stored)

 

Ingestion
  • Source to target mapping
  • Source documentation (constraints, update timing, etc)

 

Transformation
  • Business rule definitions
  • Transformation rules
  • Unit tests

 

Curation
  • Data Model
  • Data Governance
  • Lineage

 

Data Quality Checks
  • Cleanliness (Null counts, Orphan records, unmatched lookup values, etc)
  • Completeness checks
  • Appropriateness checks (bounding, negative values, unicode characters, etc)

 

Analysis
  • Data Profiling
  • Aggregate rules
  • Lookup tables
  • Findings
  • Recommendations

 

Storage
  • Day 1 size requirements
  • Predicted growth
  • Archive rules
  • Disaster recovery
