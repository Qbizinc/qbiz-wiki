This checklist is to be used to ensure that all aspects of how solutions are deployed have been properly documented and implemented for client projects.  

 

Implementation 
  • Stakeholders
  • Business Use Case / Project Info

 

Deployment 
  • Historical data load
  • Scheduled run or update
  • Ad Hoc run for custom date range

 

Provisioning 
  • Roles required for:
    • Database Schemas
    • Orchestrator
  • Additional roles or grants required
  • Is Identity based access used (if so, how is it used)
  • How to gain access to code repository
  • How to gain access to scheduled job (orchestration tool)
  • Who to contact for access support

 

Orchestration and Automation
  • Login info
  • Scheduled Jobs
  • Code Repository Location
  • Team managing code
  • Who to contact for engineering support
