Tool Evaluation - dbt Semantic Layer

 

Informal Working Notes

The dbt Semantic Layer is composed of open source components in the core product and proprietary components which are part of the dbt Cloud hosted service.

https://docs.getdbt.com/blog/understanding-the-components-of-the-dbt-semantic-layer

https://docs.getdbt.com/docs/build/metrics

Good discussion about using metrics in dbt:

   https://docs.getdbt.com/blog/how-to-design-and-structure-metrics

 

Presentation from Coalesce 2022 Hands-on: the dbt Semantic Layer – it's pretty high level and not very hands-on. The slides are here.

 

Metrics can only be calculated on materialized transforms. I.e. dbt has created a table or view in the DWH.

It is currently in Public Preview, supporting only Snowflake. A Team account is required for Metadata API access.

Roadmap for the “Semantic Layer” in dbt Core is to build out the features of the metrics with things like versioning and improved developer experience; and to add Entities to Metrics. Entities will allow joining on dimensions etc.

Roadmap for the Semantic Layer in dbt Cloud primarily about increasing performance and availability and the number of supported data warehouses.

Data caching is not part of the dbt Semantic Layer product or current roadmap. No current integration with LookML and there is overlap.
